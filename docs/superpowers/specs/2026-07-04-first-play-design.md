# 猜猜团：首次打开体验改版设计

## 1. 目标

把首次打开体验从“选择一种游戏模式”改为“立即答一道题”。

主要用户是偶然打开 Chrome 扩展、只愿意随手尝试一分钟的人。成功标准是：

- 陌生用户在打开后 5 秒内看懂玩法并完成第一次点击。
- 用户无需理解模式、等级、经验、连续挑战或主题。
- 第一次正确作答发生在高成功率、低压力的环境中。

本次设计优先提高首次开玩的转化率，不以长期留存、社交传播或扩充玩法为第一目标。

## 2. 现状判断

当前产品已经具备完整题库、四种模式、三级提示、计分、连击、等级、主题、档案和分享，但首次打开时同时暴露了过多系统。

主要问题：

1. 用户还没有体验核心乐趣，就要先理解并选择四种模式。
2. 等级、XP、连续天数和档案在首次使用时没有价值，却占用首屏注意力。
3. 四种模式主要是结束条件不同，没有形成四种足够独特的决策体验。
4. 正确反馈停留约 2.2 秒，打断“看、猜、下一题”的快节奏。
5. 三级提示及其不同扣分比例增加了规则负担，但没有创造有意义的选择。
6. 首题可能从完整题库随机选取，难度和熟悉度无法保证，不适合作为产品教学。

因此，本次改版不增加任务、道具、排行榜或新模式。改进重点是减少首次决策、提高首次成功率，并加快单位时间内的正确反馈次数。

## 3. 方案选择

采用“单一核心循环”路线。

另外两种未采用的路线：

- “试玩题 + 模式大厅”能兼容当前结构，但首页仍存在视觉竞争，复杂性只是下移。
- “强化现有大厅”改动小，但依赖文案和动效弥补结构问题，不能解决首次选择成本。

选定路线的原则是：对路过用户，第一屏最有说服力的内容不是功能介绍，而是一道他马上能猜中的题。

## 4. 信息架构

### 4.1 首次首页

首屏只展示：

- 紧凑品牌标识。
- 一道难度 1、答案为两个字的 Emoji 四选一题。
- “选出答案”这一条操作提示。
- 四个答案选项。

首屏不展示：

- 四种模式入口。
- 等级、XP 和连续天数。
- 吉祥物欢迎语和介绍性文案。
- 主题、减少动态效果和档案入口。
- 沉浸模式入口。

历史玩家可以看到一个不抢占注意力的历史最佳入口；首次用户不显示该入口。

### 4.2 首次答对之后

答对首屏题后直接进入默认的“连续快猜”，不要求用户再次确认，也不在开场强调“60 秒模式”。首屏题就是首次三题教学中的第一题，不额外增加一题。

前三题承担渐进教学：

1. 第一题：明显的两字四选一题。
2. 第二题：四选一题，但干扰项更接近。
3. 第三题：首次拼字题，自动演示并锁定第一个正确字。

第四题起进入正常题库和计分节奏。计分、连击、剩余时间等信息在此时出现，避免在用户理解玩法前制造认知负担。

### 4.3 局后页面

局后页面按以下优先级展示：

1. 本局最有意义的结果，例如答对题数或新纪录。
2. “再来一局”主按钮。
3. 每日挑战入口。
4. 分享入口。
5. 历史最佳和档案入口。

闯关与无尽模式暂时从一级导航隐藏。底层模式实现保留，避免本次改版扩大为无关重构。只有当模式能提供区别于“结束条件变化”的独特策略时，才重新进入一级入口。

## 5. 核心游戏循环

目标循环：

`看懂题目 → 一次点击 → 即时确认 → 看到一点进步 → 下一题`

### 5.1 正确反馈

- 正确后约 0.7 秒进入下一题。
- 立即突出正确选项，并给出轻量视觉反馈。
- 脑洞解释显示在非阻塞区域，随当前题滑走，不要求用户关闭。
- 不使用长篇随机鼓励文案占据主要反馈位置。

### 5.2 错误反馈

- 错误选项立即变为错误状态并轻微抖动。
- 四选一题允许继续选择剩余答案。
- 拼字题清除未锁定的错误输入，保留系统已锁定的提示字符。
- 错误反馈不额外停顿 0.7 秒，不弹出说明层。

### 5.3 连击与进步

- 连续答对 3 题后才显示连击，之前不展示零值或低价值统计。
- 局内只突出一个动态目标，例如“再答对 1 题刷新最佳”。
- 分数继续作为底层结果保存，但首次局内不需要用户理解速度、倍率和提示扣分公式。

## 6. 简化提示

把三级提示改为单次操作“帮我一步”：

- 四选一：一次排除两个错误选项。
- 拼字题：一次锁定下一个正确字符。

提示只取消当前题的速度奖励，不再使用 15%、40%、80% 三档扣分。每题只允许使用一次提示。

这保留了遇到困难时的出口，同时删除提示等级、连续点击提示和隐藏扣分规则。

## 7. 题目选择

新增独立的首次题池，不直接从 500 题完整题库随机选取。

首次题池应满足：

- 难度固定为 1。
- 答案固定为两个常用汉字。
- Emoji 与答案之间是直接组合关系，不依赖影视、歌曲或网络热点知识。
- 三个干扰项长度相同、类别相近，但不能产生多解。
- 题目通过人工白名单选择，不通过运行时规则猜测“是否简单”。

首次三题在同一次安装中保持稳定顺序，避免不同用户得到不可比较的首次体验。完成首局后恢复现有的去重随机选题逻辑。

## 8. 状态与数据

玩家资料升级为 schema 2，并增加首次体验状态：

- `onboardingCompleted`: 是否完成首次三题。
- `firstAnswerLatencyMs`: 从首次题渲染完成到第一次提交答案的毫秒数；只记录第一次。

资料迁移规则保持明确：

- 没有本地资料时创建 schema 2 资料，`onboardingCompleted` 为 `false`。
- schema 1 资料若存在任意 XP、最后游玩日期或非零最佳成绩，则迁移后 `onboardingCompleted` 为 `true`。
- 其余 schema 1 资料迁移后 `onboardingCompleted` 为 `false`。

因此，真正玩过的老玩家会跳过首次三题教学，但仍进入新的“首页即题目”结构。

不增加服务器、账号、远程配置或分析 SDK。若需要衡量 5 秒目标，首版只在本地记录聚合值，避免为了单一验证指标引入基础设施。

## 9. 组件边界

保持现有 React 架构，只做聚焦调整：

- `App`：决定展示首次题、连续快猜或局后页面。
- `FirstPlayScreen`：只负责首次题展示和渐进教学，不包含档案或模式选择。
- `GameScreen`：承载正常游戏循环，支持延迟展示局内统计。
- `ResultScreen`：负责局后主次入口，不承担模式大厅功能。
- 游戏领域函数：负责首次题组、提示效果、计分和状态推进，可独立测试。
- 玩家存储：负责新字段默认值和旧资料迁移。

不把首次体验逻辑继续堆入当前 `HomeScreen`。首次可玩题是游戏状态，不是营销首页内容，应有独立边界。

## 10. 异常与降级

- 本地资料读取失败：使用默认资料并展示首次体验。
- 首次题池缺题或校验失败：回退到完整题库中符合首次题规则的题目；仍不得展示空页面。
- 浏览器存储写入失败：当前游戏继续，不阻塞答题；下次可能重新出现教学。
- 快速重复点击：提交后立即锁定当前正确选择，防止重复计分和重复推进。
- 减少动态效果开启：用颜色和边框状态替代抖动、滑动与庆祝动画。
- 用户退出首局：返回时继续显示可作答题，不展示模式大厅。

## 11. 可访问性

- 每个 Emoji 题目提供不泄露答案的可访问标签，例如“由两个表情组成的谜题”。
- 正确、错误状态除颜色外还使用图标或文字。
- 所有选项保持键盘可达，并提供清晰焦点样式。
- 动画不阻塞操作；减少动态效果设置继续生效。
- 首屏按钮保持至少 44×44 CSS 像素的点击区域。

## 12. 验收标准

### 首次体验

- 新资料打开应用时，第一屏存在一道可直接作答的题。
- 第一屏不出现模式选择、等级、XP、连续天数、主题或档案。
- 用户通过一次点击即可提交第一题答案。
- 首次三题依次覆盖简单四选一、较近干扰项四选一和引导拼字。
- 第四题前不显示剩余时间、分数或连击。

### 反馈与提示

- 正确后约 0.7 秒自动进入下一题。
- 错误后可立即重试，不存在固定等待。
- 每题提示最多使用一次。
- 四选一提示排除两个错误项；拼字提示锁定下一个正确字。
- 使用提示只取消速度奖励。

### 信息架构

- 局后“再来一局”为唯一主按钮。
- 每日挑战可从局后访问。
- 闯关与无尽不再占据一级入口。
- 老玩家资料可以正常加载，并跳过首次三题教学。

### 工程质量

- 领域测试覆盖首次题组、状态推进、一次性提示和新计分规则。
- 存储测试覆盖新资料默认值和旧资料兼容。
- 组件测试覆盖首次点击、错误重试、第三题拼字引导和局后入口。
- TypeScript 类型检查、现有测试和生产构建全部通过。
- 在 Chrome Side Panel 尺寸下完成键盘与减少动态效果检查。

## 13. 明确不做

本次不做：

- 新模式、新道具、新任务和排行榜。
- 在线账号、云同步、服务器题库或 AI 动态出题。
- 大规模视觉品牌重做。
- 重写 500 道题或调整全部题目难度。
- 删除现有模式底层代码。
- 为首次转化指标接入第三方分析平台。

这些内容都不能直接证明“首次打开后 5 秒内开始作答”，因此不应进入本次范围。
